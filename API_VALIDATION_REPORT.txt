ROOT CAUSE ANALYSIS
================================================================================

The error "Host not in allowlist" indicates one of:

1. FASTIFY-CHECK-HOST PLUGIN (Most Likely)
   - Fastify has a built-in security plugin that validates HTTP Host header
   - Prevents Host Header Injection attacks
   - Configuration likely missing or not deployed

2. AWS APPLICATION LOAD BALANCER (ALB)
   - ALB could be checking Host header against allowlist
   - Configuration in Security Group or Target Group

3. AWS WAF (Web Application Firewall)
   - WAF rule might be blocking non-whitelisted hosts
   - Could be checking Host, Origin, or Referer headers

4. NGINX REVERSE PROXY
   - If deployed behind NGINX, host validation could be configured there
