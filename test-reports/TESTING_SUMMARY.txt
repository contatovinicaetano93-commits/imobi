ARCHITECTURE OVERVIEW
================================================================================

Frontend Component (Client-side):
  - React functional component
  - useState for: status, uploading, error, loading
  - useEffect for: initializing status on mount
  - Error boundary: try-catch in loadStatus() and handleUpload()
  - Styling: Tailwind CSS with responsive classes

API Integration:
  - Base URL: http://localhost:4000 (configurable)
  - Authentication: credentials: 'include' (cookie-based)
  - Content-Type: application/json
  - Error handling: Custom ApiError class

State Flow:
  1. Component mounts → loading=true
  2. Fetch status → GET /api/v1/kyc/status
  3. Receive data → update status state
  4. User uploads → POST /api/v1/kyc/upload
  5. Refresh status → automatically update UI

Document Types:
  - RG (Carteira de Identidade)
  - Selfie (with document in frame)

Status Values:
  - NENHUM: No documents submitted
  - PENDENTE: Awaiting review
  - APROVADO: Approved
  - REJEITADO: Rejected

API ENDPOINTS DOCUMENTED
================================================================================

GET /api/v1/kyc/status
  Response: {
    usuarioId: string,
    status: string,
    documentos: KycDocumento[],
    resumo: { pendentes: number, aprovados: number, rejeitados: number }
  }

POST /api/v1/kyc/upload
  Body: { tipo: string, url: string }
  Response: { kycDocumentoId, tipo, url, status, criadoEm, ... }

GET /api/v1/kyc/documentos
  Response: KycDocumento[]

GET /api/v1/kyc/verificar
  Response: { completo: boolean, documentos: KycDocumento[] }
