1. CLIENT COMPONENT
   - Type: "use client" (Client-side rendering)
   - Framework: React with Next.js 14

2. IMPORTS
   - useEffect, useState from React
   - kycApi types and functions from @/lib/api

3. STATE MANAGEMENT
   ├─ status: KycStatus | null
   │  └─ Holds complete KYC status including documents and summary
   ├─ uploading: boolean
   │  └─ Tracks upload operation state
   ├─ error: string | null
   │  └─ Error message display
   └─ loading: boolean
      └─ Initial page load state

4. LIFECYCLE
   - useEffect on mount → loadStatus()
   - Automatically fetches KYC status when component loads

5. KEY FUNCTIONS

   a) loadStatus()
      - Calls: kycApi.obterStatus()
      - Sets: status state, error on failure, loading=false
      - Runs: On mount and after upload

   b) handleUpload(tipo: string)
      - Type: "RG" or "Selfie"
      - Flow:
        1. Set uploading=true, clear error
        2. Generate mock S3 URL: https://s3.example.com/kyc/{tipo}-{timestamp}.jpg
        3. Call: kycApi.uploadDocumento(tipo, url)
        4. Refresh status via loadStatus()
        5. Handle errors with try-catch
      - Note: Currently uses mock URL - needs real S3 upload integration

   c) getStatusBadge(status: string)
      - Returns Tailwind CSS classes for status styling
      - Mappings:
        * PENDENTE → yellow background
        * APROVADO → green background  
        * REJEITADO → red background
        * Default → gray background

API INTEGRATION DETAILS
===============================================

File: /home/user/imobi/apps/web/lib/api.ts

1. API BASE URL
   - NEXT_PUBLIC_API_URL (env var) or default: http://localhost:4000

2. FETCH WRAPPER (apiFetch)
   - Sets Content-Type: application/json
   - Uses credentials: 'include' (for cookies)
   - Response handling: JSON parsing, error throwing
   - Status code 204 returns undefined

3. KYC ENDPOINTS

   a) obterStatus()
      - Method: GET
      - Path: /api/v1/kyc/status
      - Returns: KycStatus
      
   b) uploadDocumento(tipo, url)
      - Method: POST
      - Path: /api/v1/kyc/upload
      - Body: { tipo: string, url: string }
      - Returns: KycDocumento
      
   c) listarDocumentos()
      - Method: GET
      - Path: /api/v1/kyc/documentos
      - Returns: KycDocumento[]
      
   d) verificarKycCompleto()
      - Method: GET
      - Path: /api/v1/kyc/verificar
      - Returns: { completo: boolean, documentos: KycDocumento[] }

4. TYPE DEFINITIONS

   KycDocumento:
   {
     kycDocumentoId: string      // UUID
     tipo: string                // "RG", "Selfie", etc.
     url: string                 // S3 URL
     status: string              // "PENDENTE", "APROVADO", "REJEITADO"
     motivo_rejeicao?: string    // Optional rejection reason
     analisadoEm?: string        // ISO 8601 timestamp
     criadoEm: string            // ISO 8601 timestamp
   }

   KycStatus:
   {
     usuarioId: string
     status: string              // Overall status
     documentos: KycDocumento[]
     resumo: {
       pendentes: number
       aprovados: number
       rejeitados: number
     }
   }
